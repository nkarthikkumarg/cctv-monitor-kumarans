Remote & Tunnel Access
How this local server is made accessible to the central console and authorised staff.
  • go2rtc — live camera stream bridge (WebRTC / MJPEG)
  • Caddy — local reverse proxy on port 8080
  • ffmpeg & WebRTC libraries — codec support
  • All services restart automatically on reboot
  • Point the tunnel origin to http://127.0.0.1:8080
  • Use the public hostname for the Dashboard URL and Refresh URL fields above
  • Keep the API key confidential — regenerate it from the central console if compromised
🔒
Security Reminder Before Going Live
Review these before activating continuous monitoring.
  • Change the default admin password in Notification Settings → Admin
  • Confirm NVR IP addresses are reachable from this server
  • Ensure the Cloudflare Tunnel is active before expecting central sync
  • Limit physical access to this monitoring server
  • All camera credentials in the import file are stored encrypted at rest
Once you click Activate Monitoring, the system will begin polling all imported cameras every 10 seconds and alerting the security desk on any offline event.
